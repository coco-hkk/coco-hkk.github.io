Chrome 启动参数

chrome 启动参数可参考网站 https://peter.sh/experiments/chromium-command-line-switches/

调试、环境隔离

  • --remote-debugging-port=9527 在指定端口上启用 HTTP 远程调试
  • --user-data-dir="/path/to/Data" 重定向数据存储目录，如用户配置文件。可以隔离多个浏览器环境，chrome 多开。
  • --disk-cache-dir="/path/to/Cache" 重定向缓存存储目录。chrome 多开。
  • --disk-cache-size 自定义缓存最大值（单位 byte）。
  • --media-cache-size 自定义多媒体缓存足底安置（单位 byte）。

网络摄像头

  • --use-fake-device-for-media-stream 使用假设备进行 MediaStream 替换实际的摄像头和麦克风。虚拟网络摄像头
  • --use-file-for-fake-video-capture="视频地址 mjpeg 或 y4m 格式" 使用 .y4m 文件作为网络摄像头播放。也可实用 .mjpeg 格式

安全

  • –disable-popup-blocking 禁用弹出拦截
  • --no-sandbox 取消沙盒模式
  • --enable-leak-detection 加载网页检测泄露
  • --enable-potentially-annoying-security-features 各种安全优化
  • -incognito 启动进入隐身模式
  • --disable-popup-blocking 禁用弹出拦截
  • --disable-javascript 禁用JavaScript
  • --disable-java 禁用Java
  • --disable-plugins 禁用插件
  • --disable-images 禁用图像

窗口管理

  • --start-maximized 最大化窗口
  • --kiosk 启用 kiosk 模式。（全屏浏览模式，无法使用右键，无法使用 F11 和 F12，只能通过 ALT + F4 关闭）

更新

  • --disable-component-update 禁止组件更新
  • --disable-background-networking 禁用后台联网检查更新
  • --force-update-remote-url="0.0.0.0" 强制更新远程 URL
  • --google-url="0.0.0.0"
  • --*-*-url="0.0.0.0" 强制指向无效地址

网络和同步

  • --enable-sync 启用同步
  • --proxy-pac-url 使用 pac 代理 [via 1/2]

进程控制

  • --single-process 单进程运行
  • --process-per-tab 每个标签使用单独进程
  • --process-per-site 每个站点使用单独进程
  • --in-process-plugins 插件不启用单独进程

配置

  1. 查看 Chrome 版本信息：chrome://version。包括 Chrome 版本、操作系统信息、JavaScript 版本、用户代理、命令行参数、可执行文件路径、个人资料路径。

小技巧

  1. 在 更多工具 -> 为窗口命名 可为当前打开的浏览器命名（在 windows 任务栏中可看到命名的名字），重启之后重置。
