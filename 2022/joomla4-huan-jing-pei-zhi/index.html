Joomla4 环境搭建

joomla! 是一套开源的内容管理系统（简称 CMS）。可以把它 理解为一套免费、好用的建站系统。和它同样知名的还有 Drupal 以及 WordPress系统。

Joomla 核心优势就在于各种类型的内容发布管理，包括文字、图片、音频视频、文件等。 无论是企业、政府、教育、金融等各行业的官网制作，或是电商网站、个人博客或是内部资 讯平台 Joomla 都能胜任。

joomla 运行环境参考这里。

  • VirtualBox 6.1.34
  • Ubuntu Server 22.04
  • Apache 2.4+
  • PHP 8.0
  • MySQL 5.6+

Ubuntu Server

获取 Ubuntu Server ，在 VirtualBox 中按步骤安装。

  1. 替换默认官网源为清华镜像源 https://mirrors.tuna.tsinghua.edu.cn/ubuntu/。
  2. 勾选 openssh-server 安装 ssh 服务。

完成安装后登录，使用命令ifconfig查看 IP，结果发现是 10.0.2.15 在本地无法 ping 通。打开本地 cmd 输入指令 ipconfig 得到 以太网适配器 VirtualBox Host-Only Network 的 IP 为 192.168.56.1，且可以 ping 通。

使用 putty 远程登录 192.168.56.1 报错 Network error: connection refused。 主要原因是 Virtualbox 网络配置网卡连接方式为 网络地址转换(NAT)，需要在 端口转发规则 中设置，如下：

名称 协议 主机 IP 主机端口 子系统 IP 子系统端口
ssh TCP 192.168.56.1 22 10.0.2.15 22

现在，可以在本地通过 putty 访问 Ubuntu Server 了。

Apache

Apache 相关信息在这里查看。

  1. Ubuntu 安装 Apache2

    # 安装
sudo apt install apache2 -y

# 如果有 UFW 防火墙运行，需要允许 Apache
sudo ufw allow 'Apache'

# 查看运行状态
sudo systemctl status apache2

# 若没运行
sudo systemctl enable apache2
sudo systemctl start apache2

  2. 查看 Apache2 版本

    # 版本为 2.4.52
apache2 -v

  3. 修改默认网站根目录

    # 创建新根目录 /home/www
sudo mkdir /home/www

# 由于 apache2 属于用户和用户组都为 www-data 启动，所以要修改权限
# 或直接为用户 www-data 创建一个 home 目录
sudo chown www-data:www-data /home/www

# 修改 apache2.conf 配置文件
# 找到 /var/www/ 更改为自定义新根目录 /home/www
sudo vi /etc/apache2/apache2.conf

# 修改 000-default.conf 配置文件
#找到 DocumentRoot 后面的 /var/www/html 更改为/home/www
sudo vi /etc/apache2/sites-available/000-default.conf

#重启 apache2 服务，重启完后
sudo systemctl restart apache2

  4. 在 VirtualBox 网络设置 端口转发规则 增加 80 端口和 443 端口，这样就可以在本 地浏览器输入 192.168.56.1 访问服务器了。

PHP

PHP 相关信息在这里查看。

  1. 配置独立最新软件源

    sudo apt install software-properties-common
sudo add-apt-repository ppa:ondrej/php

  2. 安装 PHP8.1

    # 为 Apache 安装 PHP8.1 和相应模块
sudo apt install php8.1 libapache2-mod-php8.1 libapache2-mod-fcgid

# 启动模块，可在/etc/apache2/mods-enabled 中查看
sudo a2enmod php8.1 fcgid

sudo apt install php8.1-{curl,mysql,xml,fpm}
# 开启配置，可在/etc/apache2/conf-enabled 中查看
sudo a2enconf php8.1-fpm

  3. 重启 apache

  4. 在 /home/www 根目录下创建 index.php，内容为

    <?php
phpinfo();

    若显示 PHP 详细信息则表示成功。

MySQL

MySQL 相关信息在这里查看。

  1. 安装 MySQL

    sudo apt install mysql-server

  2. 登录超级用户

    # 超级用户账号密码在配置文件 /etc/mysql/debian.cnf 中保存
mysql -u debian-sys-maint -p

  3. 创建新用户并授予整个数据库所有权限

    create user 'naruto'@'localhost' identified by '123456';

grant all privileges on *.* to 'naruto'@'localhost';

安装 joomla

下载joomla ZIP 压缩包，本文中使用 4.1.2。

使用 PSFTP 工具上传到服务器，使用 7zip 解压到 /home/www 目录中并修改所有者 和组为 www-data。

在网页中打开服务器地址 192.168.56.1 即可看到 joomla 安装页面。按照步骤完成。

  1. 在数据库配置这部分，需要和 MySQL 小节中保持一致。数据库按照默认给出的创建，则 无法连接数据库。

    # 以 hkk 用户身份登录创建 joomla_db 数据库
create database joomla_db;

  2. 为了更好的确保系统兼容，需要对 PHP 进行深入设置。

    sudo apt install php8.1-gd php8.1-zip php8.1-intl

    关闭输出缓冲，在文件 /etc/php/8.1/fpm/php.ini 中注释掉 output_buffering 即可。

Win10 XAMPP 搭建

XAMPP（Apache MySQL PHP PERL）是一个功能强大的建站集成软件包，支持 Windows、 Linux 和 MacOS。官网 https://www.apachefriends.org/index.html 下载 Windows 版本。

  1. 下载并安装 XAMPP 8.1.5。
  2. 打开 XAMPP Control Panel 启动 Apache 和 MySQL 模块。
  3. 删除 xampp/htdocs 内容。
  4. 下载 joomla4 4.1.3 并解压到 xampp/htdocs 中。
  5. 打开浏览器输入localhost，即可打开 joomla 安装界面。

安装 joomla：

  1. 进入安装界面，依次选择安装语言、设置网站名称、设置登录数据。
  2. 设置数据库连接：
    • 打开网址 localhost/phpmyadmin 进入 phpMyAdmin 数据库管理界面。无需输入密 码即可登录。
    • 创建数据库 joomla_db 和 joomla 设置的数据库名一致。
    • 数据库用户名为 root，密码为空。
  3. 安装完成。

与系统兼容的 PHP 设置：

  1. 打开 Apache 模块的 config 配置选择 php.ini。
  2. 关闭（注释）output_buffering=4096。
  3. 打开（关闭注释）extension=gd 和 extension=intl。

手动删除 joomla 目录中 installation 目录。

Ubuntu XAMPP + Nginx 搭建

安装 XAMPP

在官网 https://www.apachefriends.org/download.html 获取 Linux 版本地址，使用 wget 下载。

  1. 下载 xampp-linux-x64-8.1.5-0-installer.run。

  2. 由于服务器没有图形界面，所以命令行安装。

    wget https://www.apachefriends.org/xampp-files/8.1.5/xampp-linux-x64-8.1.5-0-installer.run
sudo chmod a+x xampp-linux-x64-8.1.5-0-installer
sudo ./xampp-linux-x64-8.1.5-0-installer

# 直接 Y 或回车，默认安装位置是 /opt/lampp
# 重启 xampp
sudo /opt/lampp/lampp restart

# 查看 xampp 状态
sudo /opt/lampp/lampp status

  3. xampp 配置文档

    Apache   文档根目录：/opt/lampp/htdocs/
Apache     配置文件：/opt/lampp/etc/httpd.conf
MySQL      配置文件：/opt/lampp/etc/my.cnf
PHP        配置文件：/opt/lampp/etc/php.ini
ProFTPD    配置文件：/opt/lampp/etc/proftpd.conf
PHPMyadmin 配置文件：/opt/lampp/phpmyadmin/config.inc.php

  4. xampp 命令

    启动/停止/重启 Apache：/opt/lampp/lampp start/stop/restart(包括启动/停止/重启了 mysql,proftpd)
安全设置：/opt/lampp/lampp  security
使用 php 版本/查看版本：/opt/lampp/lampp php4(5)/phpstatus
只启动和停止 Apache：/opt/lampp/lampp startapache/stopapache
只启动和停止 MySQL：/opt/lampp/lampp startmysql/stopmysql
只启动和停止 ProFTPD 服务器：/opt/lampp/lampp startftp/stopftp
启动和停止 Apache 的 SSL 支持：/opt/lampp/lampp startssl/stopssl
卸载 XAMPP：rm –rf /opt/lampp

配置 XAMPP

  1. 由于服务器之前已经安装 Nginx 服务器，且将 80 端口和 443 端口占用，所以启动 Apache 服务失败，报错： Another web server is already running.。

    • 查看端口占用情况 netstat -tulpn | grep --color :80
    • 经过分析是由于 lampp 启动脚本内部将 80 和 443 端口号写死导致，分别修改（任意） 为 81 和 1443。
    • 配置 lampp/etc/httpd.conf，将监听端口号修改为 81 和上面保持一致。
    • 配置 lampp/etc/extra/httpd-ssl.conf，将 SSL 监听端口号修改为 1443。
    • 重启服务。打开浏览器输入 ip:81 即可打开 XAMPP 欢迎页面。

  2. 在网页打开 phpmyadmin 提示权限不足，在 /opt/lampp/etc/extra/httpd-xampp.conf 重 新配置。

    <Directory "/opt/lampp/phpmyadmin">
    AllowOverride AuthConfig Limit
    Allow from all
    Require local
    Require all granted
    ErrorDocument 403 /error/XAMPP_FORBIDDEN.html.var
</Directory>

    重启 xampp。打开 phpmyadmin 发现任何人都可以登录，需要继续配置。

    # 默认 yes 即可
sudo /opt/lampp/lampp security

# 使 phpmyadmin 可读
sudo chmod 775 /opt/lampp/phpmyadmin

    配置 /opt/lampp/phpmyadmin/config.inc.php，修改 $cfgServers[$i]['auth_type'] 为 'cookie'。

  3. xampp 开机自启

    sudo ln -s /opt/lampp/lampp /etc/init.d/lampp
sudo update-rc.d -f lampp defaults

安装 Joomla

  1. 删除 htdocs 目录内容，将官网 Joomla 安装文件上传到 htdocs 目录并解压。在网页 中打开按步骤安装 Joomla。

  2. 没有足够的权限创建 configuration.php。

    安装 XAMPP 用 root 权限，下载拷贝 Joomla 用普通用户权限，设置 httpd 的 User 和 Group 为普通用户权限。 参考这里。

Nginx 与 Apache 代理配置

Nginx 和 Apache 搭配，Nginx 负责代理，Apache 负责 Joomla 站点。

# Nginx
server {
    listen 80;

    location / {
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_pass http://127.0.0.1:800;
    }
}
# Apache
Listen 127.0.0.1:800

DocumentRoot "/opt/lampp/htdocs"
<Directory "/opt/lampp/htdocs">
    Options Indexes FollowSymLinks ExecCGI Includes
    AllowOverride All
    Require all granted
</Directory>
