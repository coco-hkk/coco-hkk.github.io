Joomla4 配置

Joomla 是一个开源内容管理系统（CMS），用于构建网站和在线应用程序。 它是免费且可扩 展的，分为前端模板和后端模板（管理员）。 Joomla 是使用 PHP，面向对象编程，软件设计 模式和 MySQL（用于存储数据）开发的。

参考：https://www.wenjiangs.com/doc/ky08ybdn

基础

Joomla 有几个重要的部分，模板、菜单、模块、组件、文章。

  • 模板：可以决定网站的风格，如页面布局、呈现方式等。改变模板时，需要重新配置。
  • 菜单：页面中导航标签由菜单实现。
  • 文章：页面具体内容由文章展示。
  • 模块：由多个菜单项组合成一个模块。有的组件也可以当做模块。
  • 组件：功能性的模块，网页主要内容生产者，可由它组合而成。文章管理、分类管理、菜 单管理也是组件。

文章管理 & 分类管理

文章管理主要负责具体的内容生产。

创建文章可使用系统自带的编辑器编辑，和写 Word 文档一样。文章支持各种内容形式，如文 本、图片、视频、链接等。

分类管理主要负责将写好的文章分类管理。

总的来说：

  1. 文章可以理解为一篇 Word 文档，可支持各种内容形式。分类管理可将文章分组管理， 一篇文章只能分到一个组中。即： 一篇或多篇文章 <--> 一个分组。

菜单项 & 菜单管理

用户访问网站是通过点击导航栏上的菜单项实现。

若想呈现各种各样的页面，菜单项必须和组件联系起来，这个过程的实现，专业术语称为 菜单项类型。

菜单项类型有如，嵌入页面、文章管理、新闻源、智能搜索、标签管理等等各种组件，每一 项又有许多子项组件。

菜单管理即是将各种菜单项分组管理，并且可将菜单链接到指定一个或多个模块。

总的来说：

  1. 菜单项联系了最基本的页面内容，由菜单项类型实现。它既可以是文章、图片、视频、 搜索等，又可以是整个或部分页面。
  2. 菜单是一组菜单项的集合，一个菜单项必然只属于某个菜单。而一个菜单可以链接一个 或多个模块，但一个模块只能绑定一个菜单。即： 一个或多个菜单项 <---> 一个菜单 <---> 一个或多个模块。

模块

模块有许多类型，如内嵌页面、在线用户、广告展示、文章管理相关、新闻相关等，以及菜 单显示和自定义。

模块作为页面布局的基本单位，用于展示内容，根据不同模板的布局指定不同位置。

配置

查看模板

模板既有前台又有后台，决定了前台和后台的页面布局，模板也可以不止有一个。

若要查看模板，必须将 Global Configuration -> Templates -> Preview Module Positions 开启。

打开浏览器输入 http://your-server-ip/?tp=1 查看模板。

隐藏 index.php 重写 url

在后台打开系统全局配置，在站点 SEO 配置。

  • 开启 Search Engine Friendly 。SEF 是 Search Engine Friendly 的缩写，即「搜寻 引擎友善」，用于优化 HTML 便于爬虫抓取。
  • 开启 Use URL Rewriting。选择服务器，重写 URL 以满足特定条件。。Apache 服务器只 需将 joomla 根目录的htaccess.txt 改为 .htaccess。
  • 打开 Add Suffix to URL。根据文档类型添加 URL 的后缀。
  • 打开 Unicode Aliases。它在音译和 Unicode 别名之间进行选择。

Joomla API Token

Joomla API Token 可使用户利用 APP 远程访问服务器。只有用户自己可以在账户信息查看 或激活 API Token，其他用户都不能查看，Super Users 只有激活或关闭权限。

若找不到 Joomla API Token，则在 System Dashboard -> Manage -> Plugins 搜索关键 字 Token 找到插件 API Authentication - Web Services Joomla Token 和 User - Joomla API Token。

  1. 确保这两个插件激活。
  2. 配置 User - Joomla API Token 的 Allowed User Groups，确保将用户所在组包含 在内。
  3. 回到用户管理，重新激活用户，即可生成并激活 Joomla API Token。

可使用 postman 测试 api 检查 Token 是否正常。

扩展

扩展即使插件或组件，可在系统配置扩展中安装。joomla 给我们提供了四种安装扩展方式， 既可以在线安装，也可以从本地上传安装。

Akeeba Backup or Joolma!

这个组件用于备份网站数据。

恢复网站数据则需要工具 kickstart 实现。

  1. 将备份文件和 kickstart 压缩文件放在网站根目录中，解压 kickstart 获取kickstart.php。
  2. 在浏览器中打开 kickstart.php，按照要求恢复备份文件。
  3. 恢复过程中需要网站超级管理员账号和密码，需要数据库账号和密码。

恢复数据时特别慢。

robots.txt

robots.txt 是一个纯文本文件，可声明网站中不想被搜索引擎访问的部分，或只收录指定 的内容。

当搜索引擎访问一个站点时，它会先检查站点根目录下的 robots.txt，若存在就按照文件 中的内容来确定访问的范围；若不存在，那么搜索机器人就沿着链接抓取。

robots.txt 作用

  1. 引导搜索引擎、蜘蛛抓取指定栏目或内容
  2. 网站改版或者 URL 重写优化时候屏蔽对搜索引擎不友好的链接
  3. 屏蔽死链接、404 错误页面
  4. 屏蔽无内容、无价值页面
  5. 屏蔽重复页面，如评论页、搜索结果页
  6. 屏蔽任何不想被收录的页面
  7. 引导蜘蛛抓取网站地图

robots.txt 语法

三个语法和两个通配符。

  1. User-agent:，语法，定义搜索引擎

    User-agent: *，定义所有搜索引擎 User-agent: Googlebot，定义谷歌，只允许谷歌蜘蛛爬取

    不同的搜索引擎的搜索机器人有不同的名称，谷歌:Googlebot、百度:Baiduspider、 MSN:MSNbot、Yahoo:Slurp。

  2. Disallow:，语法，用来定义禁止蜘蛛爬取的页面或目录

    Disallow: /，禁止蜘蛛爬取网站的所有目录 “/” 表示根目录下 Disallow: /admin，禁止蜘蛛爬取 admin 目录 Disallow: /abc.html，禁止蜘蛛爬去 abc.html 页面

  3. Allow:，语法，用来定义允许蜘蛛爬取的页面或子目录

    Allow: /admin/test/，允许蜘蛛爬取 admin 下的 test 目录 Allow: /admin/abc.html，允许蜘蛛爬去 admin 目录中的 abc.html 页面

  4. $，通配符，匹配 URL 结尾的字符

  5. *，通配符，匹配 0 个或多个任意字符

注意事项

  1. robots.txt 文件必须放在网站的根目录。

  2. robots.txt 文件名命名必须小写，记得在 robot 面加s。

  3. User-agent、Allow、Disallow 的 : 后面有一个字符的空格。

  4. 路径后面加斜杠 / 和不加斜杠的是有区别的

    Disallow: /help 禁止蜘蛛访问 /help.html、/helpabc.html、/help/index.html

    Disallow: /help/　 禁止蜘蛛访问 /help/index.html。 但允许访问 /help.html、/helpabc.html

  5. Disallow 与 Allow 行的顺序是有意义的

    允许蜘蛛访问 /admin/ 目录下的 seo 文件夹

    User-agent: * Allow: /admin/seo/ Disallow: /admin/

    如果 Allow 和 Disallow 的顺序调换一下

    User-agent: * Disallow: /admin/ Allow: /admin/seo/

    蜘蛛就无法访问到 /admin/ 目录下的 seo 文件夹，因为第一个 Disallow: /admin/ 已匹配成功。

robots meta

robots.txt 主要限制整个站点或者目录的搜索引擎访问情况，而 Robots Meta 标签则主要 针对具体页面。和其他的 META 标签（如使用的语言、页面的描述、关键词等）一样， Robots Meta 标签也是放在页面中，专门用来告诉搜索引擎 ROBOTS 如何抓取该页的内容。

Robots Meta 标签中没有大小写之分，name="Robots" 表示所有的搜索引擎，可以针对某 个具体搜索引擎（如 google）写为 name="Googlebot"，content 部分有四个指令选项： index、noindex、follow、nofollow，指令间以 “,” 分隔。

  1. Index，告诉搜索机器人抓取该页面
  2. NoIndex，告诉搜索引擎不允许抓取这个页面
  3. Follow，搜索机器人可以沿着该页面上的链接继续抓取下去
  4. NoFollow，告诉搜索引擎不允许从此页找到链接、拒绝其继续访问

指令的四种组合。

<!-- 可以抓取本页，而且可以顺着本页继续索引别的链接，缺省值 -->
<meta name="robots" content="index,follow">
<!-- 等价于 -->
<meta name="robots" content="all">

<!-- 不许抓取本页，但是可以顺着本页抓取索引别的链接 -->
<meta name="robots" content="noindex,follow">

<!-- 可以抓取本页，但是不许顺着本页抓取索引别的链接 -->
<neta name="robots" content="index,nofollow">

<!-- 不许抓取本页，也不许顺着本页抓取索引别的链接 -->
<meta name="robots" content="noindex,nofollow">
<!-- 等价于 -->
<meta name="robots" content="none">
