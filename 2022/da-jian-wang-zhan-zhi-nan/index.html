网站搭建指南

web
Web
发布日期:   2022-05-06
文章字数:   1.9k
阅读时长:   6 分
阅读次数:  

如今，创建网站是超级简单的，无论是否具备相关专业知识，只要依照正确的方法，选择合 适的平台和正确工具，一切问题迎刃而解。

创建网站

创建网站基本流程为：

  1. 注册域名
  2. 购买虚拟主机
  3. 设计网页
  4. 验证和测试网站
  5. 优化

而其中设计网页的方法有三种：

  1. 从零开始创建
  2. 使用内容管理系统(CMS)
  3. 使用网站构建器创建

注册域名

域名是网站或博客最重要的标识，一个好的域名对 SEO 优化至关重要。可以通过域名注册 商，以年度合同或长期合同方式注册域名。

  1. 选择权威域名注册服务商

    国内如，万网/阿里云、西部数码、新网互联、美橙互联、三五互联。国外如： GoDaddy、Namecheap、Domain.com。

    总之，注册域名时，尽可能选择获得 ICANN 和 CNNIC 授权认证的域名注册服务商。

  2. 选择域名原则

    • 简单好记，不要复杂。
    • 尽可能选择和网站业务相关。
    • 不要使用特殊字符。
    • 使用.com域名后缀。
    • 检查域名是否有黑历史，使用site和domain指令。

    你也可以使用域名生成器生成域名：

    • LeanDomainSearch

  3. 域名备案

    主要针对国内网站用户群体，一般建议选择国内服务器。主要优势：有利于网站收录， 国内服务器访问速度更快，用户更加信任。

  4. 泛域名解析

    域名主要是为了便于记忆服务器 IP 地址，所以顾名思义域名解析就是将域名翻译成服 务器 IP 地址。而域名又分为顶级域名、一级域名和二级域名。

    • 顶级域名：域名最右边的字符组，分为两类，一类是顶级域，如.com、.org、 .net等；还有一类是地区顶级域，如.cn等。
    • 一级域名：是由字符串 + 域名后缀组成，如：baidu.com。
    • 二级域名：又称为子域名，带有一级域名下的主机名，如：cloud.baidu.com

    回到主题，泛域名解析时利用通配符*来做二级域名以实现所有二级域名均指向同一个 顶级域名。在顶级域名前添加任何的子域名，均访问的是顶级域名地址。

购买虚拟主机

虚拟主机即是虚拟服务器、云服务器，一些大型公司拥有并管理其 WEB 服务器，其它企业 则从云服务器供应商那里租赁（这更加便宜和容易）。

国内云服务器供应商如，阿里云、腾讯等。国外如，virmach、racknerd等。

通常云服务器供应商提供了各种托管服务，可根据自身业务定制网站。

设计网页

设计和创建网页可以通过聘请 WEB 开发人员实现。对于 DIY 者，以下是三种简单方法。

从零开始

所需技能和工具：

  • 知识：HTML、CSS、JavaScript、MySQL 或 PostgreSQL
  • 集成开发环境 IDE：Sublime Text、Atom、Adobe Dreamweaver
  • FTP：FileZilla、WinSCP、Cyberduck
  1. 安装配置 IDE 本地开发环境。
  2. 规划网站结构和导航系统。 必须了解如何提供内容，导航菜单的数量，列数或内容字段 数，要包含的图像数量以及位置。使用 Photoshop 创建网页的粗略绘图。可能需要为不 同页面制作不同草稿页。
  3. 使用 HTML 和 CSS 对步骤 2 的设计进行编码。
  4. 使用 JavaScript 和 jQuery 使网站动态化。
  5. 使用 FTP 客户端将本地文件上传到服务器。

使用 CMS

所需技能和工具：

  • 知识：HTML、CSS、PHP
  • CMS：WordPress 和 Weebly、joomla、Drupal

需要学习不同 CMS 的模块，或深入自己深入定制。

使用网站构建器

所需技能和工具：

  • 工具：zyro、Wix 和 Weebly

即使不了解 Web 编程，设计者也可以通过 Drag&Drop 轻松建立一个网站。

验证和测试网站

网站准备就绪后，就可以在主要浏览器（Chrome，Firefox，Safari，Microsoft Edge，IE 11 等）以及不同屏幕尺寸上验证并测试其效果。

在线免费测试工具：

  1. 测试网页是否满足 W3C 标准：https://validator.w3.org/
  2. 浏览器适配测试：https://browsershots.org/

优化

  1. 优化网站速度
  2. 提高网站搜索的可见性，SEO 优化
  3. 使用 HTTPS
  4. 添加基本页面，如索引（主页）、关于页面和联系页面
  5. 添加图标，以下是一些免费的图标生成网站：
    • https://www.favicon-generator.org/
    • http://www.favicon.cc/
    • http://tools.dynamicdrive.com/favicon/
    • http://www.genfavicon.com/
    • http://www.favicongenerator.com/
    • https://www.logaster.com/favicon/

Web 服务器

Apache、Nginx、Caddy、Tomcat 和 Nodejs 辨析。

Apache

Apache 是指 Apache 软件基金会的 Apache HTTP Server。它能够接收 http 请求，然后返 回各类资源，如 HTML，PDF，MP4 等。它的特点是只是接收请求，然后返回资源，至于这个 资源是怎么来的，它不管，所以很多时候，大家会说 Apache 支持静态资源，动态资源 需要其他模块来支持，比如使用 CGI，比如 tomcat。

Nginx

Nginx 功能同 Apache，也是 HTTP Server，但是 Nginx 经过性能优化（使用 epoll（Linux2.6 内 核）和 kqueue（FreeBSD）网络 I/O 模型），可以支持比 apache（select 模型）高得多的连接 数，cpu 与内存占用更低，工作中常见的是用于反向代理，负载均衡和虚拟主机。

Caddy

Caddy 类似 Nginx 常用于 web 服务器或反向代理。

Tomcat

tomcat 最主要的功能是提供 Servlet/JSP 容器，虽然它也具有 http 服务器的功能，但是在管 理功能和对静态资源的处理速度上不如专门的 http 服务器，如果性能需求不高的 web 应用， 可以尝试单独使用 tomcat，但是对于高性能要求的情况，需要 http 服务器和 Tomcat 集成来完 成。

Node.js

Node.js 是一个事件驱动 I/O 服务端 JavaScript 环境，基于 Google 的 V8 引擎。node 下如 express 的应用库也是 HTTP Server 的一种，不同的是，nodejs 主要提供前端资源的服务，简 单的业务逻辑也可以在 nodejs 下完成，但是复杂的业务需要 nodejs server 再与应用服务器 进行数据交换（如 RESTful）来完成。这里面会有大前端，前后端分离之类的说法。

总结

它们都能创建 Web 服务器，但是他们关注的点不一样。

  1. Apache 通常跟 PHP 配合，但也不排除能跟 Node.js 配合反向代理。建站组合为LAMP = Linux + Apache + MySQL + PHP
  2. Nginx 通常作为反向代理，可以和 Node.js web 服务器配合。建站组合为LNMP = Linux + Nginx + MySQL + PHP
  3. Tomcat 最好跟 Java 配合。
  4. Node.js 更倾向于业务逻辑。

Apache 和 Nginx 算接近一类，node.js 和 tomcat 算接近一类。

Apache 就是静态网页服务器，将本地页面文件做一个网络映射。可以添加 mod 来扩展功能， 例如 php 模块就扩展了基于 php 的 CGI 动态页面页面能力，代理模块就是成了代理服务 器。

nginx 同 Apache，不过更多主职于代理服务器。

tomcat 就是一个 Java Servlet 容器，换个说法就是基于 java 的 CGI 动态页面服务器，静态页 面只是一个附属功能。

node.js 同样一个容器，换个说法就是基于 JavaScript 的 CGI 动态页面服务器，看上去静态页 面不算是直接功能。
