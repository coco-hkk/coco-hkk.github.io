曾买过服务器使用 wordpress 搭建网站，备过案，维护麻烦。又试用过 Maverick，但主题单 一难看。偶然发现一款 hexo 主题 matery，决定以 hexo + github 搭 建网站。

在 github 创建四个仓库。两个 private，分别是 blog-repo 和 hexo-blog-service。两 个 public，分别为 hexo-blog 和 hexo-theme-matery。

  1. blog-repo 统一管理 blog，需要保护。有两个原因：其一，我所有文档都在 blog-repo 中，不想分散管理；其二，hexo 插入图片的格式不是 markdown 标准格式，github 不 能正常显示，所以此仓库中将存储标准格式 markdown，能在 github 正常显示。
  2. hexo-blog-service 是 hexo 源码，需要保护。source/_posts 中的文档由 blog-repo 自动提交，文档中图片插入格式符合 hexo。theme/matery 主题则是本仓库的子模块， 由主题仓库 hexo-theme-matery 单独维护。
  3. hexo-blog 是网站源码。当 blog-repo 中有 png 或 md 更新时， hexo-blog-service 更新并将重新生成的网站内容推送到 hexo-blog。
  4. 最终成果，我们只需在 blog-repo 中提交文档即可，其它均由 github 自动完成。
 +---------+            +-----------------+           +---------+
 |blog-repo| --push-->  |hexo-blog-service| --push--> |hexo-blog|
 +---------+            +-----------------+           +---------+
      ^                          |pull
      |update                    v
+-----+-----+        +----------------------+
|png,md 更新|        |拉取 hexo-theme-matery|
+-----------+        +----------------------+

hexo win10 本地搭建

暂且不管 github，我们只专注把本地服务器搭建好。

基本准备

  1. 安装 nodejs （参考 这里）。
  2. 安装 git for windows （参考 这里）。
  3. 熟悉 git 及 github 基本操作。

安装 hexo

在合适位置创建目录 blog，在该目录中右键 Git Bash Here，执行以下命令：

# 安装 hexo
npm i hexo-cli -g

# 验证是否安装成功
hexo -v

初始化网站

在 blog 目录中执行以下命令以初始化网站，然后在本地打开网站。

# 初始化 blog。
hexo init

# 生成静态网页。
hexo g

# 启动本地服务器，在浏览器中打开  http://localhost:4000 即可浏览网站。
hexo s

关闭本地服务器： ctrl+c。

hexo 源码目录

介绍 blog 生成静态网页后，即运行 hexo g 后的源码目录。涉及一些后面内容。

blog/                 # hexo 源码目录。
├─node_modules/     # vodejs 组件。
├─public/           # hexo g 生成的网页源码，最终推送到 github。
├─scaffolds/        # 文章头结构模板，主要方便指令创建文章，自己写完全不需要。
├─source/           # 网页文章及布局目录，后面介绍如何生成里面内容。
│  ├─about/        # 关于页面。
│  ├─archives/     # 归档页面。
│  ├─categories/   # 分类页面。
│  ├─contact/      # 留言板页面。
│  ├─friends/      # 友情链接页面。
│  ├─tags/         # 标签页面。
│  ├─_posts/       # markdown 文章，图片。
│  ├─404/          # 404 页面
│  └─CNAME         # 域名，将 github 绑定到某个域名上。

新建页面

刚刚初始化的 blog 中，source 目录中只有 _posts 用来存放文章，还缺少许多页面。

  1. 新建分类 categories 页

    categories 页是用来展示所有分类的页面，如果在你的博客 source 目录下还没有 categories/index.md 文件，那么你就需要新建一个。

    hexo new page "categories"

    编辑你刚刚新建的页面文件 /source/categories/index.md，至少需要以下内容：

    ---
title: categories
date: 2021-01-10 14:38:00
type: "categories"
layout: "categories"
---

    以上内容为创建一个新页面的流程。

    • hexo new page "xxx" 会在 source 目录中创建一个 xxx 目录，并在该目录中创建 一个 index.md 文件。

    • index.md 内容以文件 scaffolds/page.md 为模板。我们把模板内容修改为：

      title: {{ title }}
date: 2021-01-10 14:38:00
type: ""
layout: ""

      然后手动修改 type 和 layout 字段的值即可。

    • 可见， scaffolds 目录中的模板其实可以不需要的，完全可以自己创建。不过在创建 page 时，命令确实方便。创建文章时就没必要了，只要文章头符合 hexo 要求即可。

组件 / 插件

不管是 hexo 还是主题 matery，为了实现一些功能和特效都需要安装并配置相应的组件或 插件。

hexo markdown 渲染引擎

默认安装的 markdown 引擎 hexo-renderer-marked 不能将换行转换为 空格，由于我习惯于 80 列换行，添加网址或图片时 hexo 生成的网页很难看。此 外我习惯了空行表示换行，在末尾加两个空格的换行方式真是很差劲的设计，直接导致行位 有多余空白。

# 卸载默认 markdown 引擎
npm uninstall hexo-renderer-marked

# 安装 hexo-renderer-markdown-it 引擎
npm install hexo-renderer-markdown-it --save

在 hexo 的 _config.yml 中增加 markdown 配置。

markdown:
  preset: 'default'
  render:
    html: true
    xhtmlOut: false
    breaks: false       # 关闭换行，以空行代替换行
    linkify: true
    typographer: true
    quotes: '“”‘’'
  plugins:
  anchors:
    level: 2
    collisionSuffix: ''
    permalink: false
    permalinkClass: 'header-anchor'
    permalinkSide: 'left'
    permalinkSymbol: '¶'
    case: 0
    separator: '-'

git 插件

npm i hexo-deployer-git

hexo 插入图片

markdown 默认的插入图片格式不起作用。

# 安装 hexo-asset-image
npm install https://github.com/CodeFalling/hexo-asset-image --save

在 hexo 的 _config.yml 中设置 post_asset_folder: true。

比如本文想引用图片，首先在本文所在目录创建同名目录 hexo 搭建网站，把相关图片以 png 格式放在其中，然后使用 ![测试](测试.png) 引用。不要添加相对路径，直接图片 名字。

搜索

matery 中使用到了 hexo-generator-search 的 hexo 插件来做内容搜索。

# 为主题 matery 安装搜索插件
npm install hexo-generator-search --save

在 hexo 的 _config.yml 中，新增以下的配置项：

search:
  path: search.xml
  field: post

中文链接转拼音

如果文章名称是中文的，那么 hexo 默认生成的永久链接也会有中文，这样不利于 SEO，且 gitment 评论对中文链接也不支持。

# 安装 hexo-permalink-pinyin 使生成文章时生成中文拼音的永久连接
npm i hexo-permalink-pinyin --save

在 hexo 的 _config.yml 中，新增以下的配置项：

permalink_pinyin:
  enable: true
  separator: '-' # default: '-'

文章字数统计

在文章中显示文章字数、阅读时长信息、阅读次数。

# 安装 hexo-wordcount
npm i --save hexo-wordcount

在 matery 的 _config.yml 中，将各个文章字数相关的配置激活即可：

postInfo:
  date: true # 发布日期
  update: true # 更新日期
  wordCount: true # 文章字数统计
  totalCount: true # 站点总文章字数
  min2read: true # 文章阅读时长
  readCount: true # 文章阅读次数

hexo github 搭建

本地 public 推送

先考虑，把本地的网站 blog/public 推送到 github。在 github 创建一个 public 空仓库 hexo-blog。在 hexo 的 _config.yml 中配置：

deploy:
  type: git
  repository: https://github.com/coco-hkk/hexo-blog.git
  branch: master

hexo 安装 hexo-deployer-git 插件，npm install hexo-deploy-git。

然后执行 hexo d 即可将 blog/public 推送到 hexo-blog 仓库。

此时，我们无法通过网页访问 hexo-blog，需要针对 hexo-blog 做一些设置。

  1. 在 hexo-blog 的 Settings 页面找到 GitHub Pages，在 Source 中选择刚刚推 送的 Branch: master 以及 /(root)。
  2. 在 Custom domain 中添加并保存申请的域名，例如：coco-hkk.top。
    • 购买 coco-hkk.top 域名。
    • 在 cloudfalre 添加 coco-hkk.top 站点。
    • 在 DNS 项添加记录。
      类型 名称 内容 TTL 代理状态
      A coco-hkk.top GitHub IP 自动 已代理
      CNAME www coco-hkk.github.io 自动 已代理
  3. 在 source 目录中创建文件 CNAME，内容为 coco-hkk.top。

现在，我们可以通过 https://coco-hkk.top 访问网站内容。

github 推送

本地 public 推送，是将本地编译好的网站 blog/public 推送到 hexo-blog 仓库。

现在考虑，若我们只将 blog/source/_posts 中的文章推送到 github，然后由 github 自动把 blog/public 推送到 hexo-blog，岂不美哉。

我们首先得在 github 创建一个 private 空仓库 hexo-blog-service。把它克隆到 本地。

git clone https://github.com/coco-hkk/hexo-blog-service.git

  1. 将 blog 中的内容除 theme/matery 都拷贝到 hexo-blog-service 中，然后提交。

  2. 将主题 matery 作为 hexo-blog-service 的子模块。

    cd hexo-blog-service

# 增加子模块
git submodule add https://github.com/coco-hkk/hexo-theme-matery.git themes/matery

# 提交
git add -A; git commit -m "增加 matery 子模块"; git push

为了实现自动推送，创建 github actions。在 hexo-blog-service 中创建 .github/workflows/auto_deploy.yml。

请参考 hexo-action，此 action 实现 了自动生成并推送网站功能，把它的配置文件直接拷贝到 auto_deploy.yml 即可。

需要注意一下几点：

  1. 生成公私秘钥。

    ssh-keygen -t rsa -C "username@example.com"

  2. 到 Github Pages 仓库 hexo-blog，在 Settings > Deploy Keys 随意命名一个， 把 id_rsa.pub 公钥内容保存，确保给了 写权限。

  3. 在仓库 hexo-blog-service 的 Settings > Secrets 添加名为 DEPLOY_KEY 且 内容为 id_rsa 私钥的 secret。

  4. 修改 hexo-blog-service 的配置文件 _config.yml。

    deploy:
  type: git
  repository: git@github.com:coco-hkk/hexo-blog.git
  branch: master

现在，一旦我们向私有仓库 hexo-blog-service 提交任何内容，它都会向 hexo-blog 仓库 推送生成的网页。

github 二次推送

github 推送，只要维护好仓库中 source/_posts 目录下的内容即可。但如前所述：

  1. 我有很多文档和文章存储在 github，不想把它们分散存储。
  2. hexo 在 markdown 中图片引用格式不是正确的 markdown 语法，github 不能正常显示。

所以，我新创建一个 private 仓库 blog-repo，用于存储私有文档，包括网站博客 。然后，当我把文档提交到 blog-repo 时，根据提交的内容判断是否推送到 hexo-blog-service。

  1. 生成公私钥
    ssh-keygen -t rsa -C "username@example.com"
  2. 右上角图像 Personal settings -> SSH and GPG keys -> SSH keys 新增一个 SSH key，将公钥保存，命名随意，确保给了 写权限。
  3. 在 blog-repo 的 Settings -> Secrets 新增名字为 PERSONAL_TOKEN，内容为私钥 的 secret。

为 blog-repo 新增 github actions，创建 .github/workflows/hexo-blog.yml。

name: hexo blog Push
on:
  push:
    branches:
      - master
    paths:
      - 'hexo_blog_service/**.md'
      - 'hexo_blog_service/**.png'

jobs:
  Push:
    runs-on: ubuntu-latest

    steps:
      - name: Actions checkout
        uses: actions/checkout@v1

      - name: git global config
        run: |
          git config --global core.quotepath false
          git config --local user.email "hkkdlut@sina.cn"
          git config --local user.name "coco-hkk"
          mkdir -p ~/.ssh/
          echo "${{ secrets.PERSONAL_TOKEN }}" > ~/.ssh/id_rsa
          chmod 600 ~/.ssh/id_rsa
          ssh-keyscan github.com >> ~/.ssh/known_hosts
      - name: checkout hexo-blog-service
        run: |
          cd ..
          git clone git@github.com:coco-hkk/hexo-blog-service.git hexoBlogService
      - name: Get blog-repo changes files, only for md and png.
        run: |
          cd $GITHUB_WORKSPACE
          echo "======================文件变动情况统计======================="
          # 获取变化前后所有文件的路径名，a表示前，b表示后
          diff_file_a=$(git show | grep "^diff" | awk '{print $3}')
          diff_file_b=$(git show | grep "^diff" | awk '{print $4}')
          # 变化前后的所有文件路径名放在一个变量中，遍历，区分出已删除和新增文件
          # ，修改文件名操作按照新增文件和删除旧文件计算。
          update_file="$diff_file_a $diff_file_b"
          for file in $(echo "$update_file")
          do
            file=$(echo $file | awk '{print substr($0, 3)}')
            if [ ${file:0:1} = '/' ]; then
              file=$(echo $file | awk '{print substr($0, 2)}' | sed 's/"//g')
              echo "include chinese character."
            fi
            if [ ! -f $file ]; then
              echo "X  $file"
              # deleted_file_pathx 保存已删除文件路径
              echo $file >> deleted_file_pathx
              continue
            fi
            # updated_file_pathx 保存新增文件或已修改文件名文件的路径
            echo $file | tee -a updated_file_pathx
          done
          echo "============= 统计 png 和 md 新增和更新文件路径 ==============="
          cat updated_file_pathx | uniq | grep "^1_Blog" | grep -E ".png$|.PNG$|.md$" | tee updated_file_path
          echo "=============== 统计 png 和 md 已删除文件路径 ================"
          if [ -s deleted_file_pathx ]; then
              cat deleted_file_pathx | uniq | grep "^1_Blog" | grep -E ".png$|.PNG$|.md$" | tee deleted_file_path
          fi
      - name: markdown images relative path change
        run: |
          if [ -s updated_file_path ]; then
            cat updated_file_path | grep -E ".md" | tee updated_md_file
            for file in $(cat updated_md_file)
            do
                file_name=$(basename $file .md)
                sed -E -i "s/\]\($file_name\//\]\(/g" $file
            done
          fi
      - name: update hexo-blog-service repo
        run: |
          # 先删除后拷贝，防止由于路径改变的文件被误删。
          if [ -s deleted_file_path ]; then
            echo "====================== Delete file =========================="
            cat deleted_file_path | grep ".md$" | tee deleted_md_file
            cat deleted_file_path | grep -E ".png$|.PNG$" | tee deleted_png_file
            for file in $(cat deleted_md_file)
            do
                echo  "删除 $(basename $file)"
                rm -rf ../hexoBlogService/source/_posts/$(basename $file)
            done
            for file in $(cat deleted_png_file)
            do
                png_path="../hexoBlogService/source/_posts/$(basename $(dirname $file))"
                if [ ! -d $png_path ]; then
                    continue
                fi
                echo  "删除 $(basename $file)"
                rm -rf $png_path/$(basename $file)
                # 删除空目录
                if [ "$(ls -A $png_path)" = "" ]; then
                    rm -rf $png_path
                fi
            done
          fi
          if [ -s updated_file_path ]; then
            echo "======================== Copy file =========================="
            cat updated_file_path | grep -E ".png$|.PNG$" | tee updated_png_file
            cp -rf $(cat updated_file_path | grep ".md$") ../hexoBlogService/source/_posts
            for file in $(cat updated_png_file)
            do
                png_path="../hexoBlogService/source/_posts/$(basename $(dirname $file))"
                if [ ! -d $png_path ];then
                   mkdir -p $png_path
                fi
                cp -rf $file $png_path
            done
          fi
      - name: deploy hexo-blog-service
        run: |
          cd ../hexoBlogService
          git config --local user.email "hkkdlut@sina.cn"
          git config --local user.name "coco-hkk"
          git status
          git add -A
          git commit -m "Automatically Update"
          git push --force --quiet "git@github.com:coco-hkk/hexo-blog-service.git" master:master

PS blog-repo 发现子目录 hexo_blog_service 中有 .png 和 .md 格式的文档更新， 则会自动将更新或新增的 md 文档中的图片引用替换为 hexo 可识别的格式。然后更新 hexo-blog-service 仓库并提交。

FAQ

  3. busuanzi 统计文章访问量错误

    和 Chrome 85 版本 Referrer Policy 更改有关。之前版本的 Chrome浏览器，如果网 站没有指定自己的 Referrer Policy，那么 Chrome 默认 policy 是 no-referrer-when-downgrade，在 Chrome 85 版本中，为了保护用户的隐私，默认的 Referrer Policy 则变成了 strict-origin-when-cross-origin。

    // 修改 layout.ejs
{{- if .Site.Params.busuanzi.enable -}}
  <script async src="//busuanzi.ibruce.info/busuanzi/2.3/busuanzi.pure.mini.js"></script>
  <meta name="referrer" content="no-referrer-when-downgrade">
{{- end -}}

    参考：https://jdhao.github.io/2020/10/31/busuanzi_pv_count_error/
