安装 FreeBSD

可以在虚拟机 VirtualBox 中安装或制作 U 盘启动盘在物理机上安装。从 FreeBSD 9.0 开 始，bsdinstall 代替了 sysinstall，具体教程查阅对应版本的 FreeBSD handbook。

本文是在 VirtualBox 中安装 FreeBSD。

VirtualBox 设置

VirtualBox 用户手册在 这里。

  1. 系统

    • 内存大小： 4096 MB
    • 处理器： 4
    • 启动顺序： 硬盘，光驱
    • EFI： 已启用
    • 硬件加速： VT-x/AMD-V，嵌套分页，KVM 半虚拟化

  2. 显示

    • 显存大小： 128 MB
    • 显卡控制器： VBoxSVGA（桌面显示不能设置为 VMSVGA ）

  3. 勾选 启用 EFI （只针对某些操作系统） 和 硬件时钟使用国际标准时间（UTC）

问题

  1. 开机进入 UEFI shell

    启用 VirtualBox EFI，加载 ISO 文件后，直接进入了 UEFI shell。

    在 shell 中输入 FS0: 或 FS1: 找到并执行 bootx64.efi，即可进入安装界面。

  2. Xorg 安装完成后启动报错： cannot run in framebuffer mode

    检查显卡控制器，确保不能设置为 VMSVGA 模式。

从 ISO 中安装软件

# 以 root 登录，挂载 ISO 到 dist 目录。
mkdir -p /dist
mount -t cd9660 /dev/cd0 /dist

# 导入环境变量同时安装，如安装 sudo。
env REPOS_DIR=/dist/packages/repos pkg install sudo

配置超级用户

FreeBSD 安装时创建的用户不具备超级用户权限，无法使用指令 su 切换到 root。假设 创建的用户为 naruto，将该用户加入到超级用户组中。

# 以 root 登录，修改配置文件 /etc/group
# 在 wheel 行添加 naruto 用户，wheel 组的用户成员可以使用 su 成为 root。如下：
# wheel:*:0:root,naruto

# 或使用指令
pw groupmod wheel -m naruto

也可以使用指令 sudo 临时将 naruto 用户权限提升到 root。

# 使用 root 登录，或 su 切换到 root。
pkg install sudo

# 配置 /usr/local/etc/sudoers 文件
# 将 %wheel ALL=(ALL)ALL 这一行的注释取消。

换源

  1. pkg 源

    • 禁用系统级 pkg 源

      mv /etc/pkg/FreeBSD.conf /etc/pkg/FreeBSD.conf.bak

    • 创建用户级 pkg 源

      # 创建 repos 目录，增加 taiwan 和 ustc 两个源。
mkdir -p /usr/local/etc/pkg/repos
vi /usr/local/etc/pkg/repos/taiwan.conf
vi /usr/local/etc/pkg/repos/ustc.conf
      // taiwan.conf，latest 表示最新，quarterly 表示稳定。
taiwan:{
  url: "pkg+http://pkg0.twn.freebsd.org/${ABI}/quarterly",
  mirror_type: "srv",
  signature_type: "fingerprints",
  fingerprints: "/usr/share/keys/pkg",
  enabled: yes
}
      // ustc.conf 中科大源，可将 signature_type 置为 "none"。
ustc:{
  url: "pkg+http://mirrors.ustc.edu.cn/freebsd-pkg/${ABI}/quarterly",
  mirror_type: "srv",
  signature_type: "fingerprints",
  fingerprints: "/usr/share/keys/pkg",
  enabled: yes
}

    • 更新索引以生效

      pkg udpate -f

  2. portsnap 源

    # 配置文件 /etc/portsnap.conf，修改
SERVERNAME=portsnap.cn.freebsd.org
# 其它源：portsnap.freebsd.cn, hshh.org

  3. ports 源

    # 配置文件 /etc/make.conf，若不存在则创建，增加
FETCH_CMD=axel
FETCH_BEFORE_ARGS= -n10
DISABLE_SIZE=yes
MASTER_SITE_OVERRIDE?=http://mirrors.ustc.edu.cn/freebsd-ports/distfiles/${DIST_SUBDIR}/

配置桌面

参考 这里。

  1. 安装 xorg

    # 安装 xorg
pkg install xorg
# 或通过 ISO 安装
env REPOS_DIR=/dist/packages/repos pkg install xorg

# 将使用 xorg 的用户加入到 video 或 wheel 组
pw groupmod video -m 'user' || pw groupmod wheel -m 'user'

# 在配置文件 /boot/loader.conf 中，增加
# kern.vty=vt

# 并没有在 /etc/X11 和 /usr/local/etc/X11/xorg.conf.d 中生成 xorg.conf 配置文件
# 以下为手动配置方案。虽然没有自动生成配置文件，但我也没有手动配置。
Xorg -configure
mv /root/xorg.conf.new /usr/local/etc/X11/xorg.conf.d/xorg.conf

  2. 安装 KDE5 桌面环境

    # 最小安装 plasma5-plasma，基本安装 kde-baseapps，全安装 kde5
pkg install x11/kde5
# 或
env REPOS_DIR=/dist/packages/repos pkg install x11/kde5

# KDE 依赖于 procfs，在 /etc/fstab 中添加一行
# proc    /proc     procfs    rw    0     0

  3. 安装图形驱动

    pkg install xf86-video-intel

  4. 安装 sddm 登录界面

    pkg install sddm

  5. 安装 VirtualBox ose additions 增强功能

    参考 这里。

    pkg install virtualbox-ose-additions virtualbox-ose-kmod

  6. 修改配置文件 /etc/rc.conf

    dbus_enable="YES"
hald_enable="YES"

vboxguest_enable="YES"
vboxservice_enable="YES"
vboxservice_flags="--disable-timesync"   # 若使能 ntpd 服务，则配置此项。

sddm_enable="YES"

  7. 分辨率配置

    这块挺麻烦的，我是通过重装系统侥幸没有问题，到最后也没搞明白。遇到的主要解决 方案是指令 gtf 和 xrandr，以及确保显卡控制器不能是 VMSVGA。

  8. 加速开机

    关闭 sendmail 服务。在 /etc/rc.conf 中加入 sendmail_enable=“NONE”。

本地化

FreeBSD 本地化通过用户的配置文件 ~/.login_conf 或用户 shell 的启动文件 ~/.profile、~/.bashrc 或 ~/.cshrc 设置变量 LANG 和 MM_CHARSET 实现。

  • LANG 本地化设置。格式： 语言代码_国家代码.编码。

  • MM_CHARSET 应用程序的 MIME 字符集。

    编码方式 charset 含义
    iso-8859-1 西欧的编码
    gb2312 中文编码
    utf-8 世界通用语言编码
    big5 繁体中文编码 
# 查看支持本地化的类型
locale -a

# 查看当前配置
locale

配置 .login_conf 文件：

me:\
  :charset=utf-8:\
  :lang=zh_CN.UTF-8:\
  :setenv=LC_ALL=zh_CN.UTF-8,LC_MESSAGES=zh_CN.UTF-8,LC_MONETARY=zh_CN.UTF-8,LC_NUMERIC=zh_CN.UTF-8,LC_TIME=zh_CN.UTF-8,LC_COLLATE=zh_CN.UTF-8,LC_CTYPE=zh_CN.UTF-8:

字体

# 安装字体
pkg install wqy-fonts

# 设置字体
xset fp+ /usr/local/share/fonts/wqy
xset fp rehash

中文输入法

这里有不同的输入法框架 scim、fcitx 和 ibus，选择 fcitx 安装。

pkg install zh-fcitx zh-fcitx-configtool fcitx-qt5 fcitx-m17n zh-fcitx-libpinyin

由于使用 tcsh ，故在配置文件 .cshrc 中添加：

setenv GTK_IM_MODULE fcitx
setenv GTK3_IM_MODULE fcitx
setenv XMODIFIERS @im=fcitx

若想输入法随桌面启动，则执行：

cp /usr/local/share/applications/fcitx.desktop ~/.config/autostart/

解决终端无法输入中文和显示中文：

# 编辑 .cshrc
setenv LANG zh_CN.UTF-8
setenv MM_CHARSET zh_CN.UTF-8

FreeBSD fcitx 中文输入法很麻烦，安装好后没法正常使用，打开输入法配置乱配置了很久 才能正常输入。

兼容 Linux 二进制文件

默认情况下，FreeBSD 并不开启 Linux 二进制兼容支持。开启需要加载 Linux 内核模块。

# 32 位兼容
kldload linux

# 64 位兼容
kldload linux64

# 确认 kernel 和 linux.ko 模块加载
kldstat

若想在启动时自动启用 Linux 兼容支持，可在配置文件 /etc/rc.conf 添加：

linux_enable="YES"

安装 linux_base-c7：

pkg install emulators/linux_base-c7

在配置文件 /etc/fstab 中添加：

linprocfs   /compat/linux/proc  linprocfs       rw      0       0
linsysfs    /compat/linux/sys   linsysfs        rw      0       0
tmpfs    /compat/linux/dev/shm  tmpfs   rw,mode=1777    0       0

执行

mount /compat/linux/proc
mount /compat/linux/sys
mount /compat/linux/dev/shm
