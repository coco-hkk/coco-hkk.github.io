WIN10 Emacs 安装配置。

安装 Emacs

  1. 官网 下载。

  2. 国内 镜像。

  3. 安装包

    emacs-27.1-x86_64.zip
或
emacs-27.1-x86_64-installer.exe

  4. 安装

    解压并重命名 emacs-27.1-x86_64.zip 到所需安装位置，如 D:\emacs-27.1，后面 均以此位置为例。也可以直接运行 emacs-27.1-x86_64-installer.exe。

  5. 配置 HOME 目录

  • 在注册表中添加 计算机\HKEY_LOCAL_MACHINE\SOFTWARE\GNU\Emacs 项。

  • 为 Emacs 项添加字符串值。

    名称 数据
    HOME D:\emacs-27.1

  • 增加环境变量 D:\emacs-27.1。

基本配置

配置文件

配置文件目录 .emacs.d 位置 D:\emacs-27.1\.emacs.d

  1. 可参考 purcell 的配置。

  2. 配置文件目录结构

    .eamcs.d/ 
        ├─lisp/          ;; 存放自定义配置文件
        ├─site-lisp/     ;; 存放第三方配置文件
        └─init.el        ;; 初始化配置文件

  3. 目录详解

  • lisp/

    插件配置文件目录，配置文件命名格式为 init-xxx.el

  • site-lisp/

    放置无法从 elpa 或其它仓库获取的第三方插件，本目录及其子目录须在启动时添加到 load-path

  • init.el

    初始化 load-path，初始化 elpa，加载各插件配置文件等

elpa 仓库

  1. 仓库镜像源

    • melpa
    • emacs-china
    • 清华镜像

  2. 配置文件 init-elpa.el

    • 添加仓库

      (add-to-list 'package-archives '("melpa" . "http://melpa.org/packages/") t)

    • 初始化 package

      (package-initialize)

主题配色

  1. solarized-theme

    • 主题

    • 配置文件 init-mytheme.el。

      (add-to-list 'custom-theme-load-path "your path")
(load-theme 'solarized t)

  2. molokai

    • 主题

    • 配置文件 init-mytheme.el。

      (setq molokai-theme-kit t)
(add-to-list 'custom-theme-load-path "your path")
(load-theme 'molokai t)

字体配置（解决中文卡顿）

  1. dejavu-fonts 字体

  2. 编码设置

    (set-language-environment 'UTF-8)
(set-locale-environment "UTF-8")
(set-default-coding-systems 'utf-8)

  3. 中英文等宽设置

    (defun set-font (english chinese english-size chinese-size)
   (set-face-attribute 'default nil :font
                       (format   "%s:pixelsize=%d"  english english-size))
   (dolist (charset '(kana han symbol cjk-misc bopomofo))
     (set-fontset-font (frame-parameter nil 'font) charset
                       (font-spec :family chinese :size chinese-size))))

(set-font   "Dejavu Sans Mono" "WenQuanYi Zen Hei Mono" 14 14)

插件配置

以下所有插件均可使用以下命令格式安装。

M-x package-install RET xxx RET

markdown

markdown-mode 支持 emacs markdown 编辑，效果预览需要 pandoc 工具支持。插件 markdown-toc 用于自动生成目录。

markdown-mode

  1. 下载

    • Markdown-mode
    • pandoc

  2. 配置

    (autoload 'markdown-mode "markdown-mode"
         "Major mode for editing Markdown files" t)
(add-to-list 'auto-mode-alist '("\\.text\\'" . markdown-mode))
(add-to-list 'auto-mode-alist '("\\.markdown\\'" . markdown-mode))
(add-to-list 'auto-mode-alist '("\\.md\\'" . markdown-mode))

(autoload 'gfm-mode "markdown-mode"
        "Major mode for editing GitHub Flavored Markdown files" t)
(add-to-list 'auto-mode-alist '("README\\.md\\'" . gfm-mode))

;; 需安装 pandoc
(setq markdown-command "D:/Program/Pandoc/pandoc.exe")

  3. 快捷指令

    指令 功能
    C-c C-s b 加粗
    C-c C-s i 倾斜
    C-c C-s c© code
    C-c C-s 1~6（或 h) 标题
    C-c C-s f 脚注
    C-c C-l 插入链接
    C-c C-i 插入图片
    C-c C-s C-h 查询帮助
    C-c C-s t 插入表格
    指令（with pandoc） 功能
    C-c C-c m 显示对应 html 代码
    C-c C-c p preview
    C-c C-c e export
    C-c C-c v export & preview
    C-c C-c o open
    C-c C-c C-h help

markdown-toc

  1. 下载 markdown-toc

  2. 问题

    • 标题不能以数字开头（包括中文“〇”）命名。否则，生成的目录链接无法跳转。
    • 标题必须从 h1-h6 按序开始，不能直接从其它级别开始。否则，无法正确生成目录树 。

  3. 快捷指令

    M-x markdown-toc-generate-toc ;; 生成目录树
M-x markdown-toc-refresh-toc  ;; 更新目录树
M-x markdown-toc-delete-toc   ;; 删除目录树

org 导出 md

  1. org-mode 支持 markdown 格式导出，而且不需要 pandoc 和 markdown-toc。从操作性 上来说org-mode 要好一些，功能更加强大。

  2. 配置

    (eval-after-load "org" '(require 'ox-md nil t))

  3. 导出 md

    C-c C-e m

notepad++ 预览 md

  1. notepad++ v7.6.1 可支持 markdown 预览。
  2. 插件 -> Plugins Admin…-> MarkdownViewer++

evil

模拟 vi 操作体验的 emacs 插件。

  1. 配置

    (require 'evil)
(evil-mode 1)

  2. 快捷方式

    指令 功能
    C-z emacs 模式和 evil 模式间切换

  3. 问题

    • 在 NORMAL 和 INSERT 模式之间切换时，输入法不会自动切换。
    • 一个可能的解决方案 https://emacs-china.org/t/topic/3152 但我不喜。所以， 暂时不用，什么时候官方支持什么时候用。

atuo-complete

自动补全工具。

  1. 配置

    (require 'auto-complete)
(require 'auto-complete-config)

(global-auto-complete-mode t)

;; 文本模式下启动补全功能
(add-to-list 'ac-modes 'text-mode)

(ac-config-default)

(setq ac-auto-start 4)

;; pos-tip 帮助闪烁
;;(require 'pos-tip)
;;(setq ac-quick-help-prefer-pos-tip t)

;; quick-help
(setq ac-use-quick-help t)
(setq ac-quick-help-delay 1.0)

(setq ac-dwim t)
(setq ac-trigger-commands
  (cons 'backward-delete-char-untabify ac-trigger-commands))

;; fuzzy
(setq ac-fuzzy-enable t)

;; menu
;;(setq ac-menu-height 12)
;;(set-face-background 'ac-candidate-face "lightgray")
;;(set-face-underline 'ac-candidate-face "darkgray")
;;(set-face-background 'ac-selection-face "steelblue")

  2. 问题

    • pos-tip 在这个环境下效果很差，显示的帮助内容闪烁。

    • auto-complete 插件和 company 插件冲突，在弹出菜单时可能造成显示混乱。

    • 模糊匹配 fuzzy 插件。

      (setq ac-fuzzy-enable t) ;; 不起作用
M-x ac-fuzzy-complete    ;; 可以起作用，但没用

yasnippet

为 C、C++、C#、Perl、Python 等语言设置代码片段。

  1. 配置

    (require 'yasnippet)

;; 使用 Ctrl-c k 作为唯一的触发快捷键
(define-key yas-minor-mode-map (kbd "<tab>") nil)
(define-key yas-minor-mode-map (kbd "TAB") nil)
(define-key yas-minor-mode-map (kbd "C-c k") 'yas-expand)

(setq yas-snippet-dirs
  '("~/.emacs.d/Mysnippets"))

(yas-global-mode t)

snippet

  1. 创建 snippet

    C-c & C-n (M-x yas-new-snippet) ;; 创建新 snippet
C-c C-c                         ;; 保存 snippet

  2. 典型例子

    # contributor: test <test@qq.com>
# name: __---__
# --
__${init}__

  3. # -- 以上内容均为注释，这些注释可能为指令或者元数据。

    • 指令格式类似 # property: value。
    • 若没有 # -- 则，所有的内容均被当做 snippet 内容。

  4. 指令

    • # key ：snippet abbrev，这是最重要的指令，若不指定该指令，则无法触发展开 指令。

    • # name ：snippet name，用一行描述 snippet。好的描述可以让你方便的区别相似 的 snippets。

    • # condition ：snippet condition，若 snippet 有条件，当条件非 nil 时 snippet 会被展开。

    • # group ：snippet menu grouping，一个模式若有许多 snippets，则可以把他们 组织在一个 group 中，方便菜单栏中成组的显示，防止菜单栏变得太长。

    • # expand-env ：expand environment，为变量赋值的列表（类似于 let 的变量赋 值），在展开代码段时，它可用于覆盖变量值。

      # note: 从官网说明获取，方便举例
# name: ASCII home
# expand-env: ((yas-indent-line 'fixed) (yas-wrap-around-region 'nil))
# --
welcome to my
X      humble
/ \      home,
/   \      $0
/     \
/-------\
|       |
|  +-+  |
|  | |  |
+--+-+--+

    • # binding ：direct keybinding，你可以直接将展开 snippet 的指令绑定到普通 的 emacs 按键。

    • # type ：snippet or command，若 type 指令被设置成了 command，那么 snippet 的内容会以 lisp 代码的方式执行。若设置为 snippet（默认为 snippet），snippet 内容会以其模板语法解释执行。

    • # uuid ：unique identifier，独立于 name 的唯一标识，若第二次加载的 uuid 和前次 一样，则会以第二次为准。

    • # contributor ：snippet author。

  5. 模板语法

    • 纯文本

      任意文本都可以作为模板的内容，除了美元符 $ 和反引号 ` 外，通常它们被认为 是纯文本。若需要可以用反斜杠 \ 转义美元符和反引号，若要使用双反斜杠 \\ ，则每个反斜杠需要一个反斜杠转义，即 \\\\。

    • 嵌入 elisp 代码

      使用反引号 ` 将 elisp 代码嵌入到模板中，代码会在 snippet 展开时执行。

      /*c-mode example，示例取自官网*/
#ifndef ${1:_`(upcase (file-name-nondirectory (file-name-sans-extension (buffer-file-name))))`_H_}
#define $1

$0

#endif /* $1 */

    • TAB 位字段

      该字段可以让你使用 TAB 和 S-TAB 跳转。该字段格式为美元符 $ 后面跟一个数字， $0 有着 exit point 的特殊意义，表示退出 snippet

    • 占位符字段

      制表位可以设置默认值，即占位符，语法： ${N:default value}。

    • Mirrors

      将带有占位符的制表位称为 field，一个 field 可以有许多 mirrors。

      当更新一个 field，相应 field 所有的 mirrors 都会更新

      begin {${1:test}}
$0
end{$1}

  6. snippet 实践

    # -*- mode: snippet -*-
# name: main
# key: c_main
# --
/*${1: test} begin*/
/*
C 语言 main 函数 snippet，输入 c_main 然后 C-c k 即可
*/
int main(${2:int args}, ${3:char *argv[]})
{
$0;
return 0;
}/*$1 end*/

    注：若光标被导航到了 $0，无法使用 S-TAB 向后导航了。

semx/ido

Ido 是 emacs 内置插件；semx 一款构建在 Ido 基础上，用于增强 M-x 功能的插件。

  1. ido 配置

  2. semx 配置

    (require 'smex)

(smex-initialize)

(global-set-key (kbd "M-x") 'smex)
(global-set-key (kbd "M-X") 'smex-major-mode-commands)

;; This is your old M-x.
(global-set-key (kbd "C-c C-c M-x") 'execute-extended-command)

  3. 操作

    • 左右方向键选择待选项
    • 使用 TAB 键补全

erc

登陆 emacs ERC

emacs 自带 IRC 客户端，启动指令 M-x erc。

minibuffer 会提示操作步骤：

  1. IRC server : irc.freenode.net
  2. IRC port : 6667
  3. Nickname : “your name”
  4. Password : “your passwd”

注：步骤 1-2 可按默认配置，步骤 3-4 需自定义。

注册账号

  1. 确认 Nickname

    可使用指令 /nick 修改 Nickname 至满意为止，尽量避免使用社区、商标名。

  2. 注册 nick

    /msg NickServ REGISTER password youremail@example.com （注意大小写）

freenode 会向注册邮箱发送确认注册的信息的邮件，按操作执行即可。

登陆账号

  1. 账号

    /nick "your name"

  2. 密码

    /msg NickServ identify "your passwd"

org2blog

org2blog 插件可将 emacs 中写的 org 博客推送到网站，发布博客。

  1. 安装

    M-x package-install org2blog

  2. org2blog 配置

    (maybe-require-package 'org2blog) ;; 寻找 org2blog 插件的位置，若不存在则安装
(require 'org2blog-autoloads)

;; ===============================================================
;; blog 自动登录及默认参数设置
;; 自动登录配置文件：~/.netrc  内容：machine myblog login myusername password mypassword
;; 默认参数设置 config
(require 'auth-source)
(let* ((credentials (auth-source-user-and-password "myblog"))
      (username (nth 0 credentials))
      (password (nth 1 credentials))
      (config `(("blog"
                 ;; url 你的博客中 xmlrpc.php 所在位置
                :url "http://yourWebServer/xmlrpc.php"
                :username ,username
                :password ,password
                ;; 默认发布的博客属于“未分类”
                :default-categories ("未分类")
                :keep-new-lines t
                :confirm t
                :wp-code nil
                :tags-as-categories nil))))
 (setq org2blog/wp-blog-alist config))

;; blog meta 设置，会使用上面 config 变量中设置的值
(setq org2blog/wp-buffer-template
"#+TITLE:    %s
#+AUTHOR:   %s
#+DATE:     %s
;; “目录”用中文显示
#+LANGUAGE: zh-CN
#+OPTIONS:  toc:t num:t todo:nil pri:nil tags:nil ^:nil
#+CATEGORY: %s
#+TAGS:
#+DESCRIPTION:
\n")

(defun o2b-format-function (format-string)
 (format format-string
         ;; TITLE，如果 config 中有设置 default-title 选项，则使用设置的值
         ;; 否则，需要从 minibuffer 读入用户输入的 title
         (or (plist-get (cdr org2blog/wp-blog) :default-title)
             (read-string "title: "))
         ;; AUTHOR
         user-login-name
         ;; DATE
         (format-time-string "[%Y-%m-%d %a %H:%M]" (current-time))
         ;; CATEGORY
         (mapconcat
          (lambda (cat) cat)
          (or (plist-get (cdr org2blog/wp-blog) :default-categories)
              org2blog/wp-default-categories)
          ", ")
         ))

(setq org2blog/wp-buffer-format-function 'o2b-format-function)

(provide 'init-org2blog)

  3. init.el 配置

    (require 'init-org2blog)

  4. .netrc 自动登录配置文件

    • 位置： emacs 家目录下，即~/.netrc。

    • 文件内容：

      machine myblog
login yourname
password yourpassword

  5. 操作

    • 登录 blog

      M-x org2blog/wp-login

    • 创建 blog

      M-x org2blog/wp-new-entry

    • 登出 blog

      M-x org2blog/logout

    • 切换 org2blog 模式

      M-x org2blog/wp-mode

    • 存为网站草稿

      M-x org2blog/wp-post-buffer
或
C-c M-p d

    • 发布文章

      M-x org2blog/wp-post-buffer-and-publish
或
C-c M-p p

mangit

magit 可在 emacs 中实现版本控制，思路和 git 一样，操作方式不同。

  1. 安装

    M-x package-install magit

  2. 配置

    (maybe-require-package 'magit) ;; 使 emacs 找到 magit 插件位置
(provide 'init-magit)

    在 init.el 中添加配置

    (require 'init-magit)

  3. 操作

    • 克隆远程仓库

      M-x magit-clone

    • 查看 untracked 文件

      M-x magit-status

    • 暂存 untracked 文件

      • 在 magit-status 窗口下，输入指令 s 跟踪当前文件，指令 S 跟踪所有文件。
      • 输入 ? 查看帮助

    • commit 暂存文件

      • 执行 M-x magit-commit，然后根据需要选择参数，最后 c
      • 在 magit-status 窗口下，c c 即可
      • 以上两种方式在写完 commit 信息之后，C-c C-c 提交 commit

    • push 到远程仓库

      • 执行 magit-push，选择参数，然后 p
      • 可在 magit-status 窗口执行 P，选择参数，然后 p
      • 可在 magit-stauts 窗口执行 $ 命令，打开 magit-process 查看进度

    • pull 更新

      • 执行 magit-pull, 选择参数，然后 p
      • 可在 magit-status 窗口执行 F，选择参数，然后 p

    • log 查看

      • 执行 M-x magit-log-all 可查看所有分支 log

所有操作可以在 magit-status 窗口中使用 ? 指令查看和使用。
